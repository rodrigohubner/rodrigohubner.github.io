Project applied in GSoC

Posted on ter 04 abril 2017 in misc

This year I'm applying to Google Summer of Code (GSoC). I wrote a proposal for the MNE-Python organization that stands on the umbrella of Python Software Foundation (PSF).

The MNE-python project has a rich amount of tools needed to work with EEG and MEG data. On the other hand, many BCI researchers need to work with the data at the time they are acquired and return feedback to the user. No module in MNE-Python allows that kind of online analysis yet. Thinking about this, I am proposing the idea of developing a module that allows an online analysis, initially to SSVEP-BCI (one of the most used paradigms in BCI research) using OpenBCI board (one of the most used low cost EEG equipment) for data acquisition.

Following some reasons to develop full support for online experimentation in MNE-Python:

  • Developers who like python, like to continue with python. There is a set of tools (Wyrm/Mushu/Pyff) which was developed for this purpose, but this project don't seems active.
  • OpenVibe and BCI 2000 are complete frameworks that can be used for this type of experimentation, but are written in C++, which makes it difficult for BCI researchers to create new experiments that are not offered by their interface. In addition, BCI 2000 has a non-opensource license;
  • NeuroPype (QUSP) would be the perfect alternative for python developers, however it is not opensource as well;

For these reasons, a real-time module would be essential for BCI researchers in MNE-python, which already has most of the quality cited: opensource, extensibility, flexibility, great amount of tools, very active community, etc. My idea is to develop an OpenBCI pipeline acquisition and adapt what MNE-Python already has to work with online BCI paradigms.

Initially I am studying the development of an online analysis with the SSVEP-BCI paradigm, but at the conclusion of this step, I could think of as a secondary goal the P300 paradigm as well.

As part of this project, I also want to develop a module that can receive markers for feedback using datagrams. In this way it will be possible to adapt existing feedback environments such as PyFF, scenarios using Python and Lua (OpenVibe) and even handmade scenarios with the Löve library (already built by me).

The next posts (if I am accepted) will be more focused on the GSoC project and my research in BCI.
